用户输入的表达式本身不是完整的 C 程序，例如：

`1 + 2 * 3`

它不能直接交给编译器编译。解决方法是把表达式放进一个临时函数中：

`int __expr_wrapper_1() { return 1 + 2 * 3; }`

这样表达式就变成了一段合法的 C 代码，后续就可以像普通函数一样编译成动态库，并通过 `dlsym()` 获取函数地址、调用它得到表达式结果。

---

## 1. wrapper 的作用

wrapper 本质上是一个“临时函数壳”。

它把用户输入的表达式变成：

`return 表达式;`

然后放进一个返回 `int` 的函数中。

例如用户输入：

`add(1, 2) * 3`

程序生成：

`int __expr_wrapper_1() { return add(1, 2) * 3; }`

这样后续编译器就能处理它。

---

## 2. 为什么函数名要带编号

因为用户可能输入多个表达式。

如果每次都生成同名函数，会发生重名冲突。

所以可以使用：

`__expr_wrapper_1`

`__expr_wrapper_2`

`__expr_wrapper_3`

在真正的 crepl 主循环中，可以维护一个 `expr_id`，每处理一个表达式就递增一次。

---

## 3. 需要注意的点

创建临时文件时，`mkstemps()` 的模板必须使用大写 `XXXXXX`，例如：

`/tmp/crepl-expr-XXXXXX.c`

写文件时应使用 `strlen(code)` 作为写入长度，而不是 `sizeof(code)`。

表达式后面要加分号，因为它要变成函数体里的 `return 表达式;`。
